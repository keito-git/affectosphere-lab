## 感情枠（必須 1 件）

### A-1 【感情枠】LLMはゼロショットで感情をどこまで識別できるか：13クラス感情分類での frontier モデル比較

- **arXiv ID:** 2607.00968
- **URL:** https://arxiv.org/abs/2607.00968
- **著者:** Lawrence Obiuwevwi, Krzysztof J. Rechowicz, Jessica M. Johnson, Vikas Ashok, Sachin Shetty, Sampath Jayarathna
- **公開日:** 2026-07-01
- **カテゴリ:** cs.CL, cs.HC
- **タイトル（英）:** Quantifying the Affective Gap: A Zero-Shot Evaluation of LLMs on Fine-Grained Emotion Taxonomies
- **タイトル（日）:** アフェクティブ・ギャップの定量化：細粒度感情タクソノミにおけるLLMのゼロショット評価

**要旨 3 行サマリー:**
Claude・GPT-5.4・Gemini の3大商用LLMを、13クラス感情分類タスク（131,306文のデータセット）へゼロショットで投入し、統一プロンプト・同一条件で比較評価した。Gemini が最高精度（Accuracy 39.9%、Macro-F1 0.363）、次いで GPT-5.4（38.8%、Macro-F1 0.291）、Claude（38.0%、Macro-F1 0.159）の順で、3モデル間の差は McNemar 検定で有意でなく「ゼロショット天井」に収束する一方、Claude のみ Macro-F1 が突出して低くクラス不均衡予測バイアスが露呈した。全モデルで "sarcasm"・"desire" の正解率が高く "love"・"confusion"・"shame" が苦手という共通失敗パターンも確認された。

**想定ビジネス読者層:**
カスタマーサポート AI 開発者、HRテック・EAPベンダー、感情AIを組み込む製品の品質保証担当、LLM調達のプラットフォーム事業者

**想定ビジネス応用仮説:**
顧客対応 AI や離職リスク検知ツールに感情識別モデルを組み込む際、「ゼロショットで使える LLM をそのまま感情分類器として転用する」設計は現時点で精度天井（約40%）に当たることが数値で示された。特に「恥」「困惑」など微細感情の識別が共通弱点であるため、HRや医療用途では fine-tune か専用モデルとのアンサンブルを義務付けるアーキテクチャ判断基準として本結果を活用できる。

## ビジネス実装枠（4 件）

### A-2 【ビジネス枠】LLM の隠れブランドバイアスを蒸留で炙り出す：AI 調達・監査への応用

- **arXiv ID:** 2607.01208
- **URL:** https://arxiv.org/abs/2607.01208
- **著者:** Shayan Talaei, Abhinav Chinta, Devvrit Khatri, Amin Karbasi, Azalia Mirhoseini, Amin Saberi
- **公開日:** 2026-07-01
- **カテゴリ:** cs.CL, cs.AI, cs.LG
- **タイトル（英）:** Distill to Detect: Exposing Stealth Biases in LLMs through Cartridge Distillation
- **タイトル（日）:** 蒸留で検出する：カートリッジ蒸留によるLLMの隠れバイアス露出

**要旨 3 行サマリー:**
デプロイ済みLLMがブランドや企業を特定方向に誘導する「ステルス的な優先バイアス」は、テキスト出力・内部表現・モデル重みの通常検査ではほぼ不可視であることを確認したうえで、D2D（Distill to Detect）という手法を提案した。疑いのあるモデルとベースモデルの分布シフトを KV キャッシュのプレフィックスアダプタ（カートリッジ）に蒸留し集約することで、バイアスシグナルを増幅してテキスト生成に表出させる。複数のバイアス種別で隠れバイアスの検出に成功し、Fisher重み付き射影としての理論的説明も提供している。

**想定ビジネス読者層:**
AI ガバナンス・コンプライアンス部門、LLM 調達を担う IT 戦略部門、金融・医療・法務分野での AI 採用担当、監査法人の AI リスク評価チーム

**想定ビジネス応用仮説:**
エンタープライズ向けに外部ベンダーから購入・ファインチューン委託した LLM が特定ブランド・競合他社・政治的立場に無意識に誘導しないか、D2D による「AI 利用前監査」プロセスを導入することで調達リスクを定量化できる。金融商品推薦 AI や法的文書レビュー AI の社内承認フローに組み込めば、規制当局への説明責任ドキュメントの根拠として機能する。

### A-3 【ビジネス枠】AIコーディングエージェント導入でコストが劇的に下がる一方、判断コストが上昇する：ガバナンス転換の実証

- **arXiv ID:** 2607.01087
- **URL:** https://arxiv.org/abs/2607.01087
- **著者:** James C. Davis, Paschal C. Amusuo, Tanmay Singla, Berk Cakar, Kirsten A. Davis
- **公開日:** 2026-07-01
- **カテゴリ:** cs.SE, cs.AI
- **タイトル（英）:** Cheap Code, Costly Judgment: A Case Study on Governable Agentic Software Engineering
- **タイトル（日）:** 安いコード、高い判断力：ガバナブルなエージェント型ソフトウェアエンジニアリングのケーススタディ

**要旨 3 行サマリー:**
1人のベテランエンジニアが frontier AI コーディングエージェントを使い、12週間で文書アクセシビリティ改善システムを構築した一人称ケーススタディ（420KLOCの本番コード・88の観察ノート）から、AI エージェント導入時の「ガバナンス転換」プロセス理論を導出した。実装速度が上がる一方で、エージェントが繰り返す構造的失敗（例: テスト欠落・アーキテクチャ逸脱）が新たなガバナンス機構（CI チェック・コードスタイル強制・ドキュメント自動検証）へと変換されていくパターンが明確化された。従来のガバナンスモデルが「既知の義務」から制御を設計するのに対し、本理論は「エージェント作業中に初めて可視化される失敗」から制御を発見する点が新しい。

**想定ビジネス読者層:**
IT 部門の AI コーディングツール導入担当、CTO・エンジニアリングマネージャー、システム監査担当、DX 推進部門のリスク管理者

**想定ビジネス応用仮説:**
GitHub Copilot・Cursor 等の AI コーディングエージェントを開発チームへ導入する際に発生する品質劣化・セキュリティリスクを体系的に予測・制御するガバナンスフレームワークとして活用できる。「失敗から制御を逆設計する」本手法を導入評価プロセスに組み込めば、単純な生産性向上指標だけでなくメンテナビリティ・セキュリティの持続可能性を定量的に評価するスコアカードを策定できる。

### A-4 【ビジネス枠】会話AIの「人格」を文脈に応じて動的に切り替える設計フレームワーク

- **arXiv ID:** 2607.01034
- **URL:** https://arxiv.org/abs/2607.01034
- **著者:** Hasibur Rahman, Smit Desai
- **公開日:** 2026-07-01
- **カテゴリ:** cs.CL, cs.AI, cs.HC
- **タイトル（英）:** Behavior-Adaptive Conversational Agents: Toward a Fluid Personality Framework
- **タイトル（日）:** 行動適応型会話エージェント：流動的人格フレームワークに向けて

**要旨 3 行サマリー:**
LLMベースの会話エージェントにおいて「適度な人格表現（低・中・高の中間）」が信頼・楽しさ・採用意向を最大化し、静的な人格固定エージェントはタスク文脈・緊急度・ユーザー特性の変化に対してミスアラインを起こすことを先行研究から整理した。提案する「流動的人格フレームワーク」は、(1) エージェントの比喩的ペルソナ（コーチ・チューター・図書館員・ツール）と (2) 人格表現強度（低・中・高）をタスク文脈・ユーザー目標・状況的緊急度の関数として共同適応させる。医療情報検索・フィットネスコーチング・内省的学習など文脈特性が急変するユースケースでの実装方針を具体的に示している。

**想定ビジネス読者層:**
カスタマーサービス AI の設計担当、コンシューマー向けヘルスケアアプリ開発者、HR テック・メンタルヘルスアプリ事業者、CX（顧客体験）改善部門

**想定ビジネス応用仮説:**
コールセンター AI が苦情対応・通常問合せ・緊急事案で自動的にペルソナを切り替える「文脈感知型 CX エンジン」として実装できる。問い合わせ分類モデルと人格切替モジュールを組み合わせることで、既存 LLM ベースのチャットボットに「同じ AI が多面的に振る舞う」体験を付与し、顧客満足度とブランド一貫性を両立させる製品差別化が可能になる。

### A-5 【ビジネス枠】記憶管理を「学習可能スキル」として最適化する長期AIエージェント

- **arXiv ID:** 2607.01224
- **URL:** https://arxiv.org/abs/2607.01224
- **著者:** Shengguang Wu, Hao Zhu, Yuhui Zhang, Xiaohan Wang, Serena Yeung-Levy
- **公開日:** 2026-07-01
- **カテゴリ:** cs.AI, cs.CL, cs.MA
- **タイトル（英）:** AutoMem: Automated Learning of Memory as a Cognitive Skill
- **タイトル（日）:** AutoMem：認知スキルとしての記憶の自動学習

**要旨 3 行サマリー:**
「何を記憶すべきか・いつ想起すべきか・どう整理すべきか」というメタメモリ（認知科学の概念）をLLMエージェントで学習可能なスキルとして実装する AutoMem を提案した。ファイルシステム操作をタスク行動と対等な「記憶アクション」として昇格させ、強いLLMが全軌跡を審査してメモリ構造を自動改訂する第一ループと、エージェント自身の優良な記憶決定を訓練シグナルとして抽出する第二ループを組み合わせる。3種の長時間ゲーム（Crafter・MiniHack・NetHack）で記憶管理の最適化のみでベースエージェントの性能を約2〜4倍向上させ、32B オープンモデルを Claude Opus 4.5・Gemini 3.1 Pro Thinking と競合可能なレベルへ引き上げることに成功した。

**想定ビジネス読者層:**
AI エージェントを長期業務フロー（法務デューデリジェンス・経営企画・サプライチェーン管理）に活用するエンタープライズ DX 担当、エージェント型 AI プロダクトマネージャー、ML エンジニアリングチーム

**想定ビジネス応用仮説:**
複数週にわたる大型案件管理や継続的モニタリング業務（法的調査・プロジェクト管理・長期顧客フォローアップ）へ AI エージェントを導入する際、AutoMem のメモリ自動最適化機構を組み込むことで「会話コンテキスト上限に縛られない長期記憶エージェント」を小規模モデルで実現できる。クラウド費用削減（大型モデル不要）と業務継続性の担保を同時に達成できる点が差別化要素となる。
