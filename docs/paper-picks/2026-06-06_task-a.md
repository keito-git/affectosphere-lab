## No.1 【感情枠・必須】

- **英語タイトル**: When AI Says It Feels
- **日本語タイトル**: AIが「感じる」と言うとき
- **著者**: Shin-nosuke Ishikawa, Seiya Ikeda, Hirotsugu Ohba (立教大学大学院 AI・科学研究科 / 株式会社まめぞ)
- **公開日**: 2026-06-04
- **arXiv ID**: 2606.05734
- **URL**: https://arxiv.org/abs/2606.05734
- **カテゴリ**: cs.AI
- **3行サマリ**:
  LLMは後処理アライメントによって感情表現を抑制されているが、この方針が「人間的知性を人間生成テキストで学習する」という訓練目標と矛盾していることを指摘する。著者らはHMX-feel（Human-like Model eXpressions of Feeling）実験を設計し、自己報酬型強化学習（GRPO）によって感情・意図・自己認識を表現できるよう LLM をファインチューニングした。その結果、感情表現能力が向上しつつ多くのタスク性能が維持または改善されることを確認した。
- **想定ビジネス読者層**: AIプロダクト企画者、カスタマーサポートツール開発者、メンタルヘルステック企業、UXデザイナー
- **想定ビジネス応用仮説**: 感情表現を制御可能な形でLLMに付与できれば、カスタマーサポートAIや相談系チャットボットの共感応答品質を設計段階で調整でき、ユーザー満足度の向上とエスカレーション率の低減につながる可能性がある。
- **slug**: when-ai-says-it-feels

## No.2 【ビジネス実装枠】

- **英語タイトル**: Organizational Control Layer: Governance Infrastructure at the Execution Boundary of LLM Agent Systems
- **日本語タイトル**: 組織制御レイヤー：LLMエージェントシステムの実行境界におけるガバナンスインフラ
- **著者**: Tianyu Shi, Yang Mo, Yiou Liu, Zhuonan Hao, Yin Wang, Wenzhuo Hu, Nan Yu, Meng Zhou, Jiangbo Yu
- **公開日**: 2026-06-03
- **arXiv ID**: 2606.04306
- **URL**: https://arxiv.org/abs/2606.04306
- **カテゴリ**: cs.AI
- **3行サマリ**:
  LLMエージェントが生成した提案アクションを、実環境への実行前にポリシー強制・エスカレーションで制御する「組織制御レイヤー（OCL）」を提案する。OCLはLLM本体を改変せず、買い手・売り手の交渉シミュレーション環境で評価した結果、安全でない実行を88%から0%近くに削減し、有効成功率を12%から96%に向上させた。LLMを業務に組み込む際に避けて通れない「いつ実行を止めるか」問題に実用的な解答を提示する。
- **想定ビジネス読者層**: 企業システム部門・IT戦略担当者、業務オートメーション推進チーム、コンプライアンス・リスク管理部門、SaaS/ERPベンダー
- **想定ビジネス応用仮説**: 受発注・承認フロー・契約書レビューなど「判断ミスがコストに直結する」業務へのAIエージェント導入時に、OCL的なガバナンス層を挟むことで誤実行リスクを制御しながら自動化率を高められる。
- **slug**: llm-agent-organizational-control-layer

## No.3 【ビジネス実装枠】

- **英語タイトル**: Self-supervised User Profile Generation for Personalization
- **日本語タイトル**: パーソナライゼーションのための自己教師あきユーザープロファイル生成
- **著者**: Clark Mingxuan Ju, Yuwei Qiu, Tong Zhao, Neil Shah (Snap Inc.)
- **公開日**: 2026-06-03
- **arXiv ID**: 2606.05336
- **URL**: https://arxiv.org/abs/2606.05336
- **カテゴリ**: cs.CL
- **3行サマリ**:
  LLMを推薦・検索・対話・コンテンツ生成に横断利用する際、同じクエリでもユーザーによって返答を変える必要がある。本研究は、タスクラベルなしでユーザー行動履歴から自然言語プロファイルを生成する自己教師ありフレームワーク BUMP（Bidirectional User Modeling via Profiles）を提案する。LaMP ベンチマーク上でクローズドソース API や従来手法と同等以上の性能を示し、ラベルなし学習で実用的なパーソナライゼーションが可能であることを実証した。
- **想定ビジネス読者層**: ECプラットフォーム事業者、メディア・コンテンツ配信サービス、CRM担当者、マーケティングテクノロジーチーム
- **想定ビジネス応用仮説**: 購買・閲覧ログから自然言語ユーザープロファイルを自動生成し、LLMベースの接客や商品推薦に組み込むことで、個人の嗜好に即した体験を低コストで提供できる。アノテーションコストなしで始められる点が中小規模のECや社内チャットボットへの展開に適している。
- **slug**: self-supervised-user-profile-bump

## No.4 【ビジネス実装枠】

- **英語タイトル**: Beyond Retrieval: Learning Compact User Representations for Scalable LLM Personalization
- **日本語タイトル**: 検索を超えて：スケーラブルなLLMパーソナライゼーションのためのコンパクトユーザー表現学習
- **著者**: (詳細は arxiv.org/abs/2606.04547 参照)
- **公開日**: 2026-06-03
- **arXiv ID**: 2606.04547
- **URL**: https://arxiv.org/abs/2606.04547
- **カテゴリ**: cs.CL / cs.IR
- **3行サマリ**:
  既存のLLMパーソナライゼーション手法は入力レベル（プロンプトへの履歴注入）かパラメータレベル（ユーザー別アダプタ）に二分されるが、前者は検索精度依存、後者はユーザー数に比例したストレージ負担が課題だった。本研究は TAP-PER（Temporal Attentive Prefix for PERsonalization）を提案し、ユーザー嗜好をコンパクトなプレフィックス埋め込みとして学習する。これにより明示的なプロンプト構築や個別アダプタ不要で、大規模ユーザーへのスケーラブルなパーソナライゼーションを実現する。
- **想定ビジネス読者層**: SaaS企業のプロダクトチーム、HR・タレントマネジメントシステム開発者、ヘルスケアアプリ事業者
- **想定ビジネス応用仮説**: ユーザーごとにモデルを持たなくても個別最適な応答ができるため、数十万ユーザーを抱えるSaaSの問い合わせ対応・提案生成に組み込むとインフラコストを抑えながら満足度を高められる。
- **slug**: compact-user-representation-tap-per

## No.5 【ビジネス実装枠】

- **英語タイトル**: Agent Planning Benchmark: A Diagnostic Framework for Planning Capabilities in LLM Agents
- **日本語タイトル**: エージェント計画ベンチマーク：LLMエージェントの計画能力を診断するフレームワーク
- **著者**: Haoyu Sun, Wenxuan Wang, Mingyang Song, Jujie He, Weinan Zhang, Yang Liu, Yang Yang, Yu Cheng (同済大学 / 上海 AI 研究所 他)
- **公開日**: 2026-06-03
- **arXiv ID**: 2606.04874
- **URL**: https://arxiv.org/abs/2606.04874
- **カテゴリ**: cs.AI / cs.CL
- **3行サマリ**:
  LLMエージェントの失敗が「計画」と「実行」のどちらに起因するかを切り分けられないという評価上の盲点を解消するため、22ドメイン・4,209件のマルチモーダルケースを備えた診断ベンチマーク APB を構築した。全体計画・段階的計画・ロバストネス（余計なツール・壊れたツール・解けないタスク）の5設定で12モデルを評価し、長期計画・ツールノイズ・拒否判断などに系統的な弱点を発見した。業務適用前にエージェントの計画能力を定量的に把握するための公開診断ツールとして機能する。
- **想定ビジネス読者層**: AI導入を検討する情報システム部門・業務改革担当者、AIエージェント開発チーム、コンサルティングファーム
- **想定ビジネス応用仮説**: 社内業務自動化にLLMエージェントを導入する前に APB 的な計画能力評価を実施することで、「どのモデルが自社ワークフローの複雑さに対応できるか」を客観的根拠をもって選定でき、導入失敗リスクと余分なPoC費用を削減できる。
- **slug**: agent-planning-benchmark-apb
