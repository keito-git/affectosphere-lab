### No.1 【感情枠】
- **英語タイトル**: Affective AI Safety: The Missing Piece in LLM Safety
- **日本語タイトル**: 感情的AIの安全性：LLM安全性に欠けているピース
- **著者**: Carolin Ifländer, Alba Curry, Flor Miriam Plaza-del-Arco, Amanda Cercas Curry
- **公開日**: 2026-06-22
- **arXiv ID**: 2606.23380
- **URL**: https://arxiv.org/abs/2606.23380
- **カテゴリ**: cs.CY
- **3行サマリ**: 従来のAI安全研究が誤情報・バイアス・システム信頼性など認識論的・物理的害に集中してきた一方、AIシステムが人間の感情生活に及ぼすリスクが体系的に見落とされてきたとして、「感情的安全性（affective safety）」という新概念を提案する論文。「感情的自己疎外」「公平性とバイアスの害」「関係的害」という3つの害カテゴリを分類・定義し、既存安全フレームワークがこれらに対応していないことを示す。感情的安全性には累積的・関係的・アイデンティティレベルの効果に関与する専用フレームワークが必要だと論じる。
- **想定ビジネス読者層**: AIプロダクトの安全設計・倫理審査を担う開発者・製品責任者、ユーザー向けLLMアシスタント（メンタルヘルス・教育・ケア分野）を開発するSaaS事業者、企業内AI導入のリスク管理・コンプライアンス担当者、AI規制対応を担う法務・ガバナンス部門
- **想定ビジネス応用仮説**: LLMサービスのユーザー保護設計において、従来の「有害コンテンツフィルタ」に加え感情的安全性の観点（感情的依存・関係的操作・自己疎外リスク）を組み込んだ審査・モニタリング体制を整備することで、メンタルヘルスAIや介護支援AIなど高リスクカテゴリで規制当局・ユーザーからの信頼獲得と差別化を実現できる。
- **記事スラッグ案**: affective-ai-safety-llm-emotional-harm-framework

### No.2 【ビジネス実装枠】
- **英語タイトル**: PlanBench-XL: Evaluating Long-Horizon Planning of LLM Tool-Use Agents in Large-Scale Tool Ecosystems
- **日本語タイトル**: PlanBench-XL：大規模ツールエコシステムにおけるLLMツール使用エージェントの長期計画能力評価
- **著者**: Jiayu Liu, Qihan Lin, Cheng Qian, Rui Wang, Emre Can Acikgoz, Xiaocheng Yang, Jiateng Liu, Zhenhailong Wang, Xiusi Chen, Heng Ji, Dilek Hakkani-Tür
- **公開日**: 2026-06-21
- **arXiv ID**: 2606.22388
- **URL**: https://arxiv.org/abs/2606.22388
- **カテゴリ**: cs.AI / cs.CL
- **3行サマリ**: LLMエージェントが1,665ものツールから適切なものを発見・選択しながら327件の小売タスクをこなす長期計画ベンチマーク「PlanBench-XL」を構築した研究。ツール欠損・失敗といった実世界の不確実性をシミュレートするブロッキング機構を実装し、GPT-5.4でも通常51.90%の精度がツール障害条件下で11.36%まで激減することを示した。エラーシグナルが明示されない障害や、より長い代替パスが必要な回復処理でエージェントが特に苦手とすることが明らかになった。
- **想定ビジネス読者層**: 業務自動化・RPAをLLMエージェントで置き換えを検討するIT・オペレーション部門、ERPやCRMなど多数の社内ツールをAIエージェントに操作させる基幹系DXプロジェクト担当者、AIエージェントのロバスト性評価基準を策定するベンダー・SI事業者、小売・EC事業のオペレーション自動化担当
- **想定ビジネス応用仮説**: 多数の業務ツールを横断するLLMエージェント導入前に、PlanBench-XLの評価設計（ツール障害シミュレーション付き）を自社環境に転用した「本番前ストレステスト」を実施することで、ツール連携が複雑な現場でのエージェント信頼性評価を定量化し、システム障害リスクを本番前に検出できる。
- **記事スラッグ案**: planbench-xl-llm-agent-tool-planning-retail-robustness

- **公開日**: 2026-06-22

- **想定ビジネス応用仮説**: 複数参加者が存在するビジネスチャット環境（チームMTG要約・共同作業支援・グループCS対応）にLLMを組み込む際、MuPPETの評価プロトコルを参照した「グループ会話プライバシーリスク審査」を導入することで、個人情報漏洩リスクを定量的に評価・開示でき、GDPR規制下での安全なAIアシスタント展開の根拠とできる。
- **記事スラッグ案**: muppet-llm-privacy-multiparty-conversation-enterprise

### No.4 【ビジネス実装枠】
- **英語タイトル**: VADAOrchestra: Neurosymbolic Orchestration of Adaptive Reasoning Workflows
- **日本語タイトル**: VADAOrchestra：適応型推論ワークフローのニューロシンボリックオーケストレーション
- **著者**: Teodoro Baldazzi, Luigi Bellomarini, Andrea Coletta, Michela Iezzi, Carsten Maple, Alessandro Pesare, Emanuel Sallinger
- **公開日**: 2026-06-21
- **arXiv ID**: 2606.22485
- **URL**: https://arxiv.org/abs/2606.22485
- **カテゴリ**: cs.AI / cs.CL / cs.DB / cs.LO
- **3行サマリ**: 現実の意思決定は固定シナリオに従わず、新たな文脈やデータが入るたびに推論の方向が変わるという観察に基づき、LLMオーケストレーターと記号的推論エンジン（Datalog+/-）を組み合わせたニューロシンボリックフレームワーク「VADAOrchestra」を提案した研究。LLMが動的にワークフローを計画・適応させ、その推論を論理プログラムとして符号化することで検証可能な推論追跡を実現する。金融ユースケースでの評価により、標準的なエージェントアーキテクチャ比で信頼性・拡張性・説明可能性が向上することを示した（KR 2026採択）。
- **想定ビジネス読者層**: 金融機関・保険会社での与信審査・リスク評価自動化を担当するAI活用推進担当者、コンプライアンス・法務部門でAIによる規制適合チェックを導入する企業、監査・説明責任が求められるAI意思決定システムの設計者、ERPや経営ダッシュボードに接続するインテリジェントBI基盤を構築するSI事業者
- **想定ビジネス応用仮説**: 金融機関の与信審査や不正検知において、LLMのみに依存するブラックボックス判断ではなく、VADAOrchestraのニューロシンボリック設計（論理規則+LLM）を採用することで、判断根拠を論理プログラムとして出力可能にし、規制当局への説明責任と内部監査要件を同時に満たすAI審査システムを構築できる。
- **記事スラッグ案**: vadaorchestra-neurosymbolic-reasoning-finance-compliance-ai

### No.5 【ビジネス実装枠】
- **英語タイトル**: When Does Intrinsic Self-Correction Help? A Task-Sensitive Analysis
- **日本語タイトル**: 自己修正はいつ機能するか？タスク感応型分析
- **著者**: Elroy Stav, Dvir Berlowitz, Maayan Orner, Sarit Kraus
- **公開日**: 2026-06-22
- **arXiv ID**: 2606.23196
- **URL**: https://arxiv.org/abs/2606.23196
- **カテゴリ**: cs.CL
- **3行サマリ**: LLMの「自己修正（初期回答を自分で見直して改善する能力）」が効果を発揮する条件を、タスク構造の観点から分析した研究。自己修正が機能するのは「明示的な制約の検証」「複雑な推論プロセスの再検討」「競合する戦略間の再判定」という3つのモードが活かせるタスクに限られ、それ以外では改善が見られないことを複数ベンチマーク・複数モデルで実証した。自己修正を「普遍的に信頼できる能力」ではなく「タスク固有の推論時戦略」として位置づけるべきという知見を提供する。
- **想定ビジネス読者層**: AIを活用したコードレビュー・法律文書審査・契約書チェックを構築するエンジニア・法務AIチーム、LLMを業務プロセスのQA（品質保証）ステップに組み込む製造・医療・金融業の業務改善担当者、AIエージェントのアウトプット精度改善戦略を設計するMLエンジニア・プロダクトマネージャー
- **想定ビジネス応用仮説**: LLMを用いた契約書審査・コンプライアンスチェック・医療レポート精査などの業務設計において、「自己修正が効くタスク構造（明示的制約の検証・複雑推論の再検討）」を意図的に設計することで、LLMエラーの自動検出・修正率を高め、人間によるチェックコストを削減できる。本研究の知見はどのタスクに自己修正パイプラインを適用すべきかを判断するための設計指針となる。
- **記事スラッグ案**: llm-self-correction-task-sensitive-qa-automation-enterprise
